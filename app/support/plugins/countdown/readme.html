Package Contents

  • A line folder containing the library files.
  • A presets folder containing ten unique starter presets.

If you just want to get started and don't know which files to pick, choose the tick.js file and the tick.css file. This version of Tick will automatically initialise Tick Counters on the website and can be used with the snippets on the demo site. If you use these files you can skip the next few paragraphs and jump to the Setup section.

The list below shows how the folder are structured. All Tick files are available in different formats. The tick.js file and the tick.css file are the kickstart/global versions of the files below all merged into one.

Let's begin with the CSS folder. These are the CSS files you need (again, this is only so if you don't use the tick.css variant.)

  • tick.core.min.css
  • tick.view.line.min.css

Let's move on to the JavaScript files. Depending on your project you need to pick one of the folders. The names of the folders are mostly self explanatory.

  • kickstart (will expose Tick on global window scope and automatically start looking for Tick Counters)
  • jquery (for use with jQuery)
  • global (will expose Tick on the global window scope)
  • amd (if you want to use an AMD loader)
  • commonjs (if you use a tool like Browserify)
  • module (for use with ES6 projects)

Each folder contains a tick.core.polyfill.js file which you can use if you want to support IE11.

So you've picked the files matching your project. Now let's add them to the web page. Note that the load order of the files is relevant. For JavaScript you need to load the core javascript file last. For the CSS files it's best to load the core CSS file first.

For CSS that would for example look like this (in the <head> of the document):

<link href="/lib/tick.core.min.css" rel="stylesheet">
<link href="/lib/tick.view.line.min.css" rel="stylesheet">

For JavaScript Kickstart that would look something like this (before the closing </body> tag):

<script src="/lib/tick.core.polyfill.js"></script>
<script src="/lib/tick.view.line.global.min.js"></script>

If you've bought Dots you'll also have to add the fonts as well.

<script src="/lib/tick.font.lowres.global.min.js"></script>
<script src="/lib/tick.font.highres.global.min.js"></script>

<script src="/lib/tick.core.kickstart.min.js"></script>

  1. If your using the Tick Counter jQuery API, know that it will not load counters automatically, you'll have to parse the DOM for Tick nodes yourself using the following jQuery snippet $('body').tick('parse').

  2. Again, if you need to support IE11 you'll have to add tick.core.polyfill.js to your project.

  3. If you use the commonjs, module or amd version you'll have to manually call the Tick.plugin.add method (see documentation for more details) to add plugins to Tick.

Setup

To use Tick you have to upload the files we discussed earlier to a folder on your website. For the code snippets below we'll assume it's the tick.css and tick.js version and the folder we are uploading to is named line.

Copy the following HTML snippet in the <head> of your web page. <link href="/line/tick.css" rel="stylesheet">

Then copy the following snippet and place it just before the closing </body> tag.

<script src="/line/tick.js"></script>

Make sure the paths in the above code snippets match the location of the CSS and JS files.

Presets copy pasted to your website should now function properly.

Information on how to customize Tick and use the API can be found on the product website.

  • What browser are you using to test;
  • On what OS are you testing;
  • Any screenshots that show the problem;
  • A step by step test scenario (what steps can I take to reproduce the problem);
  • If you got a public website I can test on that helps as well;

If the answer to your question is not in the FAQ and you've carefully read the documentation, contact me via the support tab on the product page and supply me with as much info as possible, I will be more than happy to help you.
